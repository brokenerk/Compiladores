What's this?

This is an attempt to implement the first half of the components of a compiler. As you can see in the next image, there are 5 basic and essential stages in any compiler:

In this web page, we implemented a lexical analyzer, and 6 parsers, that we explain in detail at the functionalities section.

Also, we offer operations with Non deterministic Finite Automatas (NFA) such as: creation, concatenation, join, positive, Kleene and optional closure. Additionally, there's the possibility to assign tokens to each automata generated, in order to convert a NFA into a Deterministic Finite Automata DFA, that will generate a lexical analyzer (through their tabular representation), and consequently, its respective parsers.

Each automata would be displayed in a graphic form like this:

This project was developed from august to december 2019 as part of the Compilers signature in ESCOM - IPN Mexico, taught by the teacher Andres Ortigoza.

Which functionalities offer and compilers topics integrate?
  • check Creation, operations and visualization of Non deterministic Finite Automatas
  • check Conversion from Non deterministic Finite Automatas to Deterministic Finite Automatas and visualization of its tabular form
  • check Lexical Analysis of Deterministic Finite Automatas through tokens
  • check Creation of Deterministic Finite Automatas and Grammars using recursive descent analysis
  • check Grammars and strings parsers using LL(1), LR(0), LR(1) and LALR analysis

How to use this?

In each section there's a button with the caption Important. If you click on it, it will display the instructions of how to enter grammars, strings, tokens and symbols, depending of the functionalities of each section.
